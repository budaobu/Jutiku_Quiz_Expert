# Quiz Data Schema Specification

本文档定义了 AI 出题专家系统的标准 JSON 输出结构，用于确保题目数据的可移植性与自动化处理。

## 1. 结构定义

```json
{
  "qid": "字符串类型，自增ID（如 ds_001）",
  "type": "字符串类型，可选：单选题 | 多选题 | 判断题",
  "lev": 数字类型，1（基础）、2（进阶）、3（专家）,
  "point": ["数组类型，包含该题考察的核心知识点标签"],
  "question": "字符串类型，题目正文",
  "options": ["A. 选项一", "B. 选项二", "C. 选项三", "D. 选项四"],
  "correct_answer": ["数组类型，正确答案字母列表"],
  "explanation": [
    "A. 针对选项A的详细正误分析",
    "B. 针对选项B的详细正误分析",
    "C. 针对选项C的详细正误分析",
    "D. 针对选项D的详细正误分析"
  ],
  "score": 数字类型，建议分值
}
```

## 2. 字段约束说明

| 字段名 | 约束条件 | 说明 |
| :--- | :--- | :--- |
| `qid` | 必须唯一 | 推荐格式：`[科目缩写]_[编号]` |
| `type` | 枚举值 | 仅限：`单选题`、`多选题`、`判断题` |
| `lev` | 整数 | 1: 概念识别；2: 综合应用；3: 逻辑推导 |
| `options` | 数组 | 判断题时固定为 `["A. 正确", "B. 错误"]` |
| `correct_answer` | 数组 | 单选题包含 1 个元素，多选题包含 >=2 个元素 |
| `explanation` | 数组 | 长度必须与 `options` 一致，按字母顺序逐一解析 |

## 3. 示例对象

```json
{
  "qid": "cs_001",
  "type": "单选题",
  "lev": 2,
  "point": ["计算机网络", "TCP协议"],
  "question": "在 TCP 三次握手过程中，第二次握手发送的报文段中，其标志位状态为？",
  "options": [
    "A. ACK=1, SYN=0",
    "B. ACK=1, SYN=1",
    "C. ACK=0, SYN=1",
    "D. RST=1, SYN=1"
  ],
  "correct_answer": ["B"],
  "explanation": [
    "A. 错误。这是第三次握手的报文特征。",
    "B. 正确。服务器收到请求后回复确认（ACK）并同步序列号（SYN）。",
    "C. 错误。第一次握手时 ACK 标志位为 0。",
    "D. 错误。RST 用于强制重置连接，不属于正常握手流程。"
  ],
  "score": 5
}
```
