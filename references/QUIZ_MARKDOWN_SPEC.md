# Markdown 出题书写规范 (Quiz Markdown Specification)

为了确保题目在 Markdown 环境下的可读性与解析一致性，请遵循以下排版协议。

---

## 1. 题目基本结构

每一道题目应包含 **题干**、**选项列表**、**元数据表格** 及 **详细解析** 四个部分。

### [题目 ID] 题目正文
> 此处填写题干内容。如涉及数学公式，请使用 $LaTeX$ 渲染。

* **A.** 选项内容
* **B.** 选项内容
* **C.** 选项内容
* **D.** 选项内容

| 属性 | 内容 |
| :--- | :--- |
| **题型** | 单选题 / 多选题 / 判断题 |
| **难度** | 1 (基础) / 2 (进阶) / 3 (专家) |
| **分值** | 数字 (如: 5) |
| **知识点** | `标签1`, `标签2` |
| **正确答案** | **[A, B]** |

#### 💡 选项解析
* **A.** 针对 A 选项的逻辑说明及正误判定。
* **B.** 针对 B 选项的逻辑说明及正误判定。
* **C.** 针对 C 选项的逻辑说明及正误判定。
* **D.** 针对 D 选项的逻辑说明及正误判定。

---

## 2. 编写细则

1.  **判断题规范：**
    * 选项固定为：`A. 正确`，`B. 错误`。
2.  **多选题规范：**
    * 正确答案需在表格中明确列出所有字母，如 `[A, C, D]`。
3.  **公式处理：**
    * 行内公式：$E=mc^2$
    * 独占行公式：
      $$ \frac{-b \pm \sqrt{b^2 - 4ac}}{2a} $$
4.  **禁止事项：**
    * 禁止在选项中使用“以上都对”或“以上都不对”，应设计具体的逻辑干扰项。
    * 禁止在 UI 相关题目中建议使用红-紫-蓝渐变色。

## 3. 示例展示

### ds_021 关于 Python 装饰器的理解
> 下列关于 Python 装饰器（Decorator）的描述中，哪一项是错误的？

* **A.** 装饰器本质上是一个闭包函数。
* **B.** 装饰器可以在不修改原函数代码的前提下增加新功能。
* **C.** 使用 `@` 语法糖是调用装饰器的唯一方式。
* **D.** 装饰器可以接收参数以实现更灵活的逻辑。

| 属性 | 内容 |
| :--- | :--- |
| **题型** | 单选题 |
| **难度** | 2 |
| **分值** | 5 |
| **知识点** | `Python`, `闭包`, `语法糖` |
| **正确答案** | **[C]** |

#### 💡 选项解析
* **A.** 正确。装饰器通常接收一个函数并返回一个增强后的函数，符合闭包定义。
* **B.** 正确。这是装饰器的核心设计目的，体现了开闭原则。
* **C.** 错误。`@` 只是语法糖，开发者完全可以通过 `func = decorator(func)` 的形式手动调用。
* **D.** 正确。通过三层嵌套函数可以实现带参数的装饰器。
